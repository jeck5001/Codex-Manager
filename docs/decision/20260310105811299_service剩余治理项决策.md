# Service 剩余治理项决策

## 结论
- `rpc_dispatch`：暂不继续拆分，当前结构已满足长期维护的边界要求。
- `gateway/protocol_adapter/mod.rs`：暂不继续拆成更细 facade/implementation 双层，当前门面已足够薄。
- `gateway/observability/http_bridge/mod.rs`：暂不继续拆成更细 facade/implementation 双层，当前门面已足够薄。
- `crates/service/tests/*` 共享支持层：暂不统一抽象，当前只有 `gateway_logs/support.rs` 具备明显共享价值，其余测试未出现高重复基座。

## 依据

### 1. `rpc_dispatch`
- 当前入口文件已经位于目录模块下：`crates/service/src/rpc_dispatch/mod.rs`
- 根层唯一调用入口仅有 `crate::handle_request -> rpc_dispatch::handle_request`
- 子职责已按 `account / apikey / app_settings / gateway / requestlog / service_config / usage` 分文件分发
- `mod.rs` 除参数读取、统一响应包装、初始化分支外，没有额外业务堆叠

决策：
- 保持当前形态，避免为了再拆一层而制造无收益的跳转成本
- 后续仅在 `initialize` 流程或统一错误包装继续膨胀时，再考虑引入 `bootstrap.rs` / `dispatch_table.rs`

### 2. `protocol_adapter/mod.rs`
- 当前职责主要是：
  - 定义 `ResponseAdapter`
  - 定义 `AdaptedGatewayRequest`
  - 将请求路由到 `request_mapping`
  - 将响应路由到 `response_conversion`
  - 管理 `prompt_cache` 环境重载
- 其内部逻辑虽然包含几段 path/protocol 判定，但规模仍在单一门面可接受范围内
- 再拆一层会引入额外模块跳转，但不会显著降低耦合，因为状态类型仍需保留在同域根入口

决策：
- 保持当前 `mod.rs` 作为协议适配总门面
- 后续仅在协议种类继续增加，或 `adapt_request_for_protocol` 出现更多分支时，再拆 `request_dispatch.rs`

### 3. `http_bridge/mod.rs`
- 当前职责主要是：
  - 装配 `aggregate / openai / delivery / stream_readers`
  - 暴露 `respond_with_upstream`
  - 暴露少量 stream reader 类型
  - 维护单个 `push_trace_id_header` 辅助函数
- `respond_with_upstream` 实现已下沉到 `delivery.rs`
- `aggregate` 和 `openai` 各自已经是独立子域

决策：
- 保持当前 `mod.rs` 为桥接门面
- 若后续 `push_trace_id_header` 之外再出现多项 header/build 辅助，届时单独抽 `headers.rs`

### 4. `tests` 共享支持层
- `gateway_logs/support.rs` 是当前唯一高价值共享测试基座，承担 `TestServer`、请求录制、上游响应拼装等完整场景能力
- 其他测试目录当前更偏局部、轻量和按子域组织，没有形成第二个可复用大基座
- 强行抽公共 support 层，容易把局部测试依赖拉高，反而削弱测试隔离性

决策：
- 保持当前测试结构
- 仅在出现第二个以上“跨多个测试文件共享的大型 fixture/server helper”时，再建立 `tests/support/`

## 对待办清单的影响
- 可将以下待办标记为已完成（已评估并决策暂不继续拆分）：
  - `rpc_dispatch` 子域边界评估
  - `protocol_adapter/http_bridge` facade 继续下沉评估
  - `service/tests` 共享支持层整理评估

## 后续触发条件
- `rpc_dispatch/mod.rs` 超过当前明显体量并新增多段初始化/错误分支
- `protocol_adapter/mod.rs` 新增新的协议族或更多 path 判定
- `http_bridge/mod.rs` 新增多项 header/build/stream glue 辅助
- `tests/` 出现第二套以上跨文件共享的大型 fixture/server support
