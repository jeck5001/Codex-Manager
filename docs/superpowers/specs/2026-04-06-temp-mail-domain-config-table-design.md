# Temp-Mail Domain Config Table Design

## Goal

把 Cloudflare Temp-Mail 域名配置区从“左侧列表 + 右侧详情”改成更接近服务列表的表格结构：上方为标准列表表格，下方为当前选中项的详情编辑区，同时保留“复制为新建”能力。

## Current Context

- 当前实现已经完成了多域名配置的数据模型和两栏编辑器。
- 用户反馈更偏好服务列表那种标准表格视觉，而不是左侧卡片列表。
- 当前每条域名配置字段较多，不适合全部塞进单个可编辑表格单元格。
- 后端接口和保存模型已经稳定，仍然是一次性提交完整 `domainConfigs` 数组。

## Recommended Approach

采用“表格列表 + 下方详情编辑”的两段式结构：

- 上半区使用和“服务列表”风格接近的 `Table` 作为域名配置列表。
- 下半区展示当前选中项的完整编辑表单。
- 表格只负责浏览、切换和触发操作，不承担复杂字段编辑。
- 详情区继续负责编辑配置字段。

这是最贴近现有“服务列表”视觉语言、同时又不把交互复杂度推高的方案。

## Why This Approach

- 表格外观直接对齐用户已经熟悉的服务列表。
- 域名配置字段数量较多，分成“列表浏览”和“详情编辑”后仍然清晰。
- 比“整表可编辑”更稳定，也更容易维护验证。
- 不需要改后端接口，只调整前端布局和选择状态。

## UI Structure

### Top Section: Domain Config Table

表格列保留：

- 配置名
- 域名后缀
- Zone ID
- 同步状态
- 操作

表格行为：

- 点击整行，选中该配置。
- 当前选中行要有明显高亮。
- 表头右上角保留 `新增域名配置`。

操作列保留：

- `编辑`
- `复制为新建`
- `删除`

其中：

- `编辑` 的行为就是选中当前行，并滚动/聚焦到下方详情编辑区。
- `复制为新建` 生成副本并自动选中新副本。
- `删除` 删除该配置，并按照既定回退规则选中下一条或上一条。

### Bottom Section: Selected Config Detail

表格下方保留完整详情表单，只显示当前选中项：

- 配置名
- Zone ID
- 域名后缀
- 子域名模式
- 随机长度
- 子域名前缀
- 同步 Cloudflare
- 要求同步成功

如果没有任何配置：

- 表格区域显示空状态提示
- 详情区也显示空状态，引导用户先新增配置

## Selection Rules

- 首次加载：
  - 如果有配置，默认选中第一条
  - 没有配置则不选中任何项
- 新增后自动选中新建项
- 复制后自动选中新副本
- 删除当前选中项时：
  - 优先选中下一条
  - 没有下一条则选上一条
  - 如果列表为空，则进入空状态
- 删除非当前选中项时，保持现有选中项不变

## Copy-As-New Behavior

复制规则保持不变：

- 复制当前项全部可编辑字段
- 生成新 `id`
- 名称非空时变为 `原名称-副本`
- 名称为空时保持空字符串
- 新副本插入到源项后方
- 复制后自动选中新副本

## Save Behavior

- 仍然复用现有 `saveCloudflareSettings`
- 保存时提交完整 `domainConfigs` 数组
- 不新增单条保存接口
- 详情区修改实时写回本地表单状态

## Refresh And Draft Behavior

保留当前已修正的草稿语义：

- 本地未保存修改存在时，不自动被远端设置覆盖
- 点击 `刷新设置` 明确表示放弃本地草稿，并重新加载远端设置

这条语义需要继续保留，避免重新引入状态覆盖问题。

## Testing Strategy

前端验证至少覆盖：

- 表格存在并显示配置摘要字段
- 点击某一行后，下方详情区切换到对应配置
- 新增后自动选中新建项
- 复制为新建后自动选中新副本
- 删除当前选中项后，选中回退规则正确
- 保存时仍然提交完整 `domainConfigs`

不需要新增后端测试，因为接口不变。

## Scope Boundaries

本次只改域名配置区的呈现和交互结构，不包含：

- 全表内联编辑
- 域名配置排序
- 搜索过滤
- 分页
- 后端协议变更
