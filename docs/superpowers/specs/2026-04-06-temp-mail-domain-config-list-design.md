# Temp-Mail Domain Config List Design

## Goal

把“Cloudflare Temp-Mail 设置”里的域名配置编辑区改成“左侧列表 + 右侧详情编辑”，并为每条域名配置提供“复制为新建”操作，方便在相同规则基础上快速派生新域名配置。

## Current Context

- 当前页面位于 `apps/src/app/email-services/page.tsx`。
- 现有 `domainConfigs` 已经是数组数据结构，后端接口也支持批量保存多条配置。
- 当前交互是把每条域名配置以内联大卡片的形式直接堆叠在页面中。
- 当配置数量增加时，页面会迅速变长，定位、切换、复用单条配置都比较低效。

## Recommended Approach

采用“两栏编辑器”方案：

- 左侧为域名配置列表，只展示摘要信息和操作入口。
- 右侧为当前选中项的完整表单。
- “新增域名配置”创建空白项并自动选中。
- “复制为新建”基于当前项生成一条新配置，自动生成新 `id`，并将名称默认改成“原名-副本”。
- 保存时仍然一次性提交整个 `domainConfigs` 数组，不改后端协议。

这是当前代码结构下最稳的方案，因为数据模型已经是列表，只需要重构前端状态与渲染方式，不需要调整接口或存储层。

## Why This Approach

- 左侧列表让用户先看到“有哪些配置”，而不是先面对大量表单字段。
- 右侧详情区适合当前这组字段密度，不会像弹窗或抽屉那样频繁打断编辑流。
- “复制为新建”放在列表项操作里，最符合“从现有配置派生新配置”的心智模型。
- 后端保持不变，风险集中在前端交互层，验证成本更低。

## UI Structure

### Left Panel

左侧列表每条显示：

- 配置名
- 域名后缀
- Zone ID
- 同步状态摘要

每条列表项提供：

- 选中当前配置
- `复制为新建`
- `删除`

列表顶部提供：

- `新增域名配置`

选中态需要明确高亮，避免用户误以为自己正在编辑另一条配置。

### Right Panel

右侧保持现有表单字段，但只展示当前选中项：

- 配置名
- Zone ID
- 域名后缀
- 子域名模式
- 随机长度
- 子域名前缀
- 同步 Cloudflare
- 同步失败是否阻止创建

当左侧没有任何配置时，右侧显示空状态说明，引导用户先新增一条。

## Selection Rules

- 页面首次加载后：
  - 如果存在配置，默认选中第一条。
  - 如果没有配置，右侧显示空状态。
- 新增配置后，自动选中新建项。
- 复制为新建后，自动选中新副本。
- 删除当前选中项后：
  - 优先选中同位置的下一条。
  - 如果没有下一条，则选中上一条。
  - 如果列表为空，进入空状态。

## Copy-As-New Behavior

“复制为新建”规则：

- 复制当前项的所有可编辑字段值。
- 生成新的 `id`。
- `name` 规则：
  - 原名称非空时，默认改成 `原名称-副本`
  - 原名称为空时，保持空字符串
- 新副本插入到原项后面，便于连续编辑。

该操作只影响前端表单状态，只有用户点击保存后才会真正写入后端。

## Save Behavior

- 仍然调用现有 `saveCloudflareSettings`。
- 提交载荷仍然是完整的 `domainConfigs` 数组。
- 右侧编辑区对当前选中项的任何修改，实时写回列表状态。
- 不新增“单条保存”语义，避免与当前接口模型冲突。

## Error Handling

- 如果当前没有选中项，右侧不渲染表单控件，避免对空对象写入。
- 删除最后一条配置后，右侧立即切换为空状态，不保留脏引用。
- 如果保存失败，保留当前本地编辑态，方便继续修正后重试。

## Accessibility And Interaction Notes

- 左侧列表项应支持整项点击切换，不只依赖小按钮。
- 列表项操作按钮要避免嵌套按钮结构，继续遵守当前页面已经使用的 shadcn/base-ui 触发器模式。
- 在移动端，两栏结构可以退化为上下布局：列表在上，详情在下。

## Testing Strategy

前端测试至少覆盖：

- 初始化时默认选中第一条配置。
- 点击列表项后，右侧切换到对应配置。
- 点击“新增域名配置”后创建空白项并自动选中。
- 点击“复制为新建”后生成新 `id`、名称加 `-副本`、其他字段保持一致，并自动选中新副本。
- 删除当前选中项后，选中项按预期回退。
- 保存时仍然提交完整 `domainConfigs` 数组。

本次不需要新增后端测试，因为接口与数据结构保持不变。

## Scope Boundaries

本次只处理域名配置编辑体验，不包含：

- 域名配置排序
- 域名配置搜索
- 域名配置分页
- 单条配置独立保存
- 后端接口重构

这些能力后续如果需要，可以在当前列表结构上继续扩展。
