## 通用验收标准

以下标准适用于所有功能：

| # | 条件 | 验证方式 |
|---|------|----------|
| G1 | 桌面端（Tauri）和 Web/Docker 模式均可正常使用 | 两种模式分别手动验证 |
| G2 | 新增数据库表有对应 migration 文件，升级不丢失现有数据 | `cargo test` migration 测试 |
| G3 | 新增 RPC 方法在 `rpc_dispatch/` 中有正确路由 | 手动 RPC 调用验证 |
| G4 | 新增设置项支持环境变量覆盖 | 设置环境变量后验证 |
| G5 | 前端新增页面/组件在移动端宽度下不溢出 | 浏览器缩放验证 |
| G6 | 网关链路新增逻辑不导致请求延迟显著增加 | 对比前后 P95 延迟 |
| G7 | `cargo clippy` 无新增 warning | CI 验证 |
| G8 | 所有新增前端文本使用中文（与现有风格一致） | 代码审查 |

## F04 请求限流

| # | 验收条件 | 通过标准 |
|---|----------|----------|
| 4.1 | 可配置 RPM 限制 | 设置 API Key RPM = 10 |
| 4.2 | RPM 限流生效 | 1 分钟内发送 15 个请求，第 11 个开始返回 429 |
| 4.3 | 429 响应规范 | 响应包含 `Retry-After` 头，body 为标准 OpenAI 错误格式 |
| 4.4 | 限流窗口重置 | 等待 1 分钟后，请求恢复正常 |
| 4.5 | TPM 限流生效 | 设置 TPM = 1000，发送大 token 请求后触发限流 |
| 4.6 | 日上限生效 | 设置日上限 = 50，超过后当日所有请求返回 429 |
| 4.7 | 未配置时不限流 | 未设置限流的 API Key 请求无影响 |
| 4.8 | 限流配置持久化 | 重启服务后限流配置不丢失 |

| 6.6 | 容量上限 | 达到最大条目数后，最旧条目被淘汰 |

| 6.8 | 手动清除 | 调用 clear 后缓存为空 |
| 6.9 | 可按 Key 开关 | 某 API Key 关闭缓存后，该 Key 请求不走缓存 |
| 6.10 | 性能影响 | 缓存查找不导致延迟增加超过 1ms |
